BUG 中文版首页 cn/index.html JS 文件路径错误
中文版首页 cn/index.html 中的 3 个 JS 文件引用路径缺少 ../ 前缀，导致在服务器上部署后 JS 完全不加载，页面交互功能（导航、轮播、WOW 动画）全部失效。CSS 文件正确使用了 ../css/，但 JS 没有同步。
cn/index.html 第 27-29 行
<!-- 当前 (错误) -->
<script defer src="js/wow.min.js"></script>
<script defer src="js/bideo.js"></script>
<script defer src="js/main.js"></script>

<!-- 应改为 -->
<script defer src="../js/wow.min.js"></script>
<script defer src="../js/bideo.js"></script>
<script defer src="../js/main.js"></script>
修复方案：将这 3 行的 src 属性从 js/ 改为 ../js/。同时检查 cn/ 目录下所有其他页面是否存在类似问题。
JS 错误 bideo.js 在所有页面报错 — 引用了不存在的 DOM 元素
bideo.js 在 IIFE 中直接执行 new Bideo() 并查找 #background_video 元素，但项目中没有任何页面包含该元素。该脚本被加载在全部 ~20 个页面上，每次打开页面都会在控制台抛出 JavaScript 错误，影响其他脚本的执行和页面性能。
js/bideo.js 第 3-6 行 + 所有 HTML 页面 <script src="js/bideo.js">
// bideo.js - 会报错，因为 #background_video 不存在
(function () {
  var bv = new Bideo();
  bv.init({
    videoEl: document.querySelector('#background_video'), // null
    container: document.querySelector('body'),
    // ... 会在 init 内部抛出 TypeError
  });
}());
修复方案：方案 A — 从所有页面中移除 bideo.js 的引用（推荐，因为该视频背景功能根本未使用）；方案 B — 在 bideo.js 中添加空值守卫：if (!document.querySelector('#background_video')) return;。
安全漏洞 jQuery 1.10.1 严重过时，存在已知 XSS 漏洞
当前使用 jQuery 1.10.1（发布于 2013 年），存在多个已公开的安全漏洞，包括 CVE-2015-9251（通过 $.parseHTML 的 XSS）、CVE-2019-11358（原型污染）、CVE-2020-11022/11023（.html() 方法中的 XSS）。对于面向客户的商业网站，这是重大安全风险。
js/jquery-1.10.1.min.js — 所有页面引用
修复方案：升级到 jQuery 3.7.1（最新稳定版）。大部分 API 兼容，需要注意：1) .bind() 已废弃，改用 .on()；2) $(window).load() 改为 $(window).on('load', ...)；3) $.browser 已移除。审查 main.js 和 web.js 中使用的 API 均兼容，升级风险低。

SEO 完全缺少结构化数据 (JSON-LD / Schema.org)
网站没有任何 application/ld+json 结构化数据。对于一个 B2B 制造商网站，缺少 Organization、Product、BreadcrumbList 等 Schema 会严重影响 Google 搜索结果中的富文本展示（如产品价格、评分、面包屑导航），直接影响搜索点击率。
修复方案：至少添加 Organization Schema（含公司名、地址、电话、邮箱、Logo）；产品详情页添加 Product Schema；所有页面添加 BreadcrumbList。
SEO 缺少 Open Graph / Twitter Card 社交分享标签
所有页面均无 og:title、og:description、og:image、twitter:card 等社交媒体元标签。当用户在微信、WhatsApp、LinkedIn 等平台分享链接时，无法生成预览卡片，严重影响社交传播效果。
修复方案：在每个页面的 <head> 中添加 OG 标签，至少包括 og:title、og:description、og:image（指向 banner 图）、og:url。
SEO 缺少 canonical URL 标签
没有 <link rel="canonical"> 标签。由于网站有中英文双语版本（/index.html 和 /cn/index.html），缺少 canonical 和 hreflang 标签可能导致搜索引擎将两个版本视为重复内容。
修复方案：英文页面添加 <link rel="canonical" href="https://www.hyteng.com/xxx.html">；中文页面添加对应 canonical；并添加 <link rel="alternate" hreflang="en" ...> 和 <link rel="alternate" hreflang="zh-CN" ...> 互相指向。
BUG about-container CSS 类未定义，about.html 布局无约束
about.html 中大量使用 <div class="about-container"> 作为内容容器，但该类在任何 CSS 文件中均未定义。导致 about 页面的所有内容区块没有最大宽度限制和居中约束，在大屏幕上内容会拉伸到整个视窗宽度。
about.html (多处) + cn/about.html (多处) + 所有 CSS 文件中搜索无结果
修复方案：在 global.css 或 about 页面内联样式中添加：.about-container { max-width: 1200px; margin: 0 auto; padding: 0 20px; box-sizing: border-box; }
编码 nav.js 文件中文注释全部乱码
js/nav.js 文件的中文注释全部显示为乱码（如 ͳһƶ��˵�������），说明文件编码与 HTML 声明的 UTF-8 不一致（可能是 GBK/GB2312 编码）。虽然不影响 JS 执行，但严重影响代码可维护性，且在某些环境下可能导致字符串字面量出错。
js/nav.js 第 1-11 行及全文注释
修复方案：用编辑器以 GBK 编码打开 nav.js，将内容复制出来，再以 UTF-8 编码重新保存。或者直接重写注释为英文/UTF-8 中文。
性能 所有图片缺少 width/height 属性和 lazy loading
全站 <img> 标签均未设置 width 和 height 属性，也未使用 loading="lazy"。前者导致浏览器在图片加载前无法预留空间，引起累积布局偏移 (CLS)，影响 Core Web Vitals 评分；后者导致首屏外的图片也立即加载，浪费带宽。
所有 HTML 文件中的 <img> 标签
修复方案：1) 为所有 <img> 添加 width 和 height 属性（浏览器会自动按比例缩放）；2) 为首屏以下的图片添加 loading="lazy"；3) 首屏图片（如 banner）不添加 lazy。
中等优先级 (Medium) 8
重复 style.css 与 global.css 存在大量重复样式定义
.content、.page-down、.banq、.smallest-screen、.blank3~blank80、.clean、.clearfix、.nav_handle 等类在 style.css 和 global.css（以及 reset.css）中被重复定义。由于 CSS 层叠规则，后加载的 global.css 会覆盖 style.css 中的同名属性，这种重复既增加文件体积，又容易导致样式不一致和维护困难。
修复方案：将 style.css 中的通用样式整合到 global.css，style.css 仅保留页面专属样式。同时清理 reset.css 中与 global.css 重复的 .blank* 等工具类。
维护 大量内联 CSS 嵌入 HTML — index.html 近 1000 行样式
index.html 在 <style> 标签中内联了约 980 行 CSS（第 34-1012 行），about.html 内联约 410 行。这些样式无法被浏览器缓存，每次页面访问都需要重新下载。且多个页面间存在相同的内联样式片段（如 Hero 渐变背景、section-header 等），修改时需要逐页同步。
修复方案：将各页面的内联 <style> 提取为独立 CSS 文件（如 css/page-index.css、css/page-about.css），在 <head> 中引用。通用的 Hero、section-header 样式合并到 global.css。
过时 reset.css 包含 IE6/IE7 专属 hack 代码
reset.css 中包含 *html（IE6 hack）、*+html（IE7 hack）、zoom:expression(...)（IE6 hasLayout hack）等远古浏览器兼容代码。这些代码在现代浏览器中完全无用，增加文件体积且影响代码可读性。
reset.css 第 93-105 行
修复方案：删除所有 *html、*+html、zoom:expression(...) 相关代码。考虑使用更现代的 CSS Reset（如 normalize.css 或自定义最小 reset）。
死代码 media.css 包含多个空的媒体查询块
media.css 中有多个空的 @media 块：@media (max-width: 1024px)、@media (max-width: 850px)、@media (max-width: 480px)、@media (max-width: 450px)、@media (max-width: 400px)、@media (max-width: 360px)，这些空块没有任何样式规则，纯属冗余。
media.css 第 18-20, 41-43, 84-86, 87-89, 90-92, 93-95 行
修复方案：删除所有空的 @media 块。
可访问性 移动端导航缺少 aria 属性和键盘可操作性
汉堡菜单按钮 .nav_handle 没有 role="button"、aria-label、aria-expanded 等无障碍属性。键盘用户无法通过 Tab 键聚焦并操作菜单。展开的子菜单也缺少 aria-haspopup 和 aria-expanded。内页导航 nyheader 中的 nav.js 有部分 aria 处理，但首页 header 完全没有。
修复方案：为 .nav_handle 添加 role="button" tabindex="0" aria-label="菜单" aria-expanded="false"，并在 JS 中同步切换 aria-expanded 状态。为 .inmuen 添加 role="navigation"。
性能 核心 JS 同步加载阻塞页面渲染
首页 index.html 中 jQuery、Swiper、SuperSlide 三个核心 JS 文件在 <head> 中同步加载（无 defer 或 async），阻塞 HTML 解析和页面首次渲染。注释说"必须同步加载"，但实际上这些库完全可以用 defer 加载，因为依赖它们的代码（main.js）已经使用了 defer。
index.html 第 22-24 行
修复方案：将三个核心 JS 都加上 defer 属性。defer 会保证执行顺序，且不阻塞渲染。这样所有 JS 都在 head 中带 defer 加载，既保持依赖顺序又不阻塞渲染。
安全 邮箱地址使用 innerHTML 注入 — 存在 XSS 模式
index.html 第 1301 行使用 document.getElementById('emailDisplay').innerHTML = ... 来混淆邮箱地址防止爬取。虽然当前代码中的字符串是硬编码的（安全），但使用 innerHTML 是不良实践，应使用 textContent 或 DOM API。此外，这种混淆方式对现代邮箱爬虫效果有限。
index.html 第 1300-1302 行
// 当前写法
document.getElementById('emailDisplay').innerHTML =
  '<a href="mailto:' + 'info' + '@' + 'hyteng.com" ...>info@hyteng.com</a>';
修复方案：改用 DOM API 构建元素：var a = document.createElement('a'); a.href = 'mailto:info@hyteng.com'; a.textContent = 'info@hyteng.com'; document.getElementById('emailDisplay').appendChild(a);
死链接 Banner 轮播图链接指向空的锚点 "#"
首页 3 个 Banner 轮播图的 <a href="#"> 链接指向空锚点，点击后页面会跳到顶部，用户体验差。这些链接应该指向具体的产品分类页或相关内容页。
index.html 第 1069-1071 行
修复方案：将 href="#" 改为实际目标页面，如 products-plug.html、products-ip44.html 等，或移除 <a> 标签只保留图片。

过时 idangerous.swiper 库已废弃
idangerous.swiper 是 Swiper 的旧名称（现已更名为 Swiper.js，由 nolimits4web 维护），版本极旧。现代 Swiper (v11) 体积更小、功能更强、支持模块化加载。
修复方案：长期可升级到 Swiper 11，API 变化较大需要适配。短期不影响功能，优先级低。
过时 Font Awesome 4.x 已停止维护
项目使用 Font Awesome 4.x（基于 fa fa-* 类名），该版本已停止维护。Font Awesome 6 提供了更多图标、更好的性能和 SVG 格式支持。
修复方案：可升级到 Font Awesome 6（类名从 fa fa-* 改为 fas fa-*），或考虑使用更轻量的替代方案如 Lucide / Heroicons SVG 图标。
不一致 JS 加载方式在各页面间不统一
首页 index.html 对非核心 JS 使用了 defer，但内页（about.html 等）的 wow.min.js 和 nav.js 既没有 defer 也没有放在 <body> 底部，而是在 <head> 中同步加载。bideo.js 在首页有 defer，在内页没有。加载策略不统一导致性能表现不一致。
修复方案：统一所有页面的 JS 加载策略：所有 <script> 放在 <head> 中并统一使用 defer 属性。
SEO 缺少 sitemap.xml 和 robots.txt
项目根目录未发现 sitemap.xml 和 robots.txt 文件。对于 50+ 页面的网站，缺少站点地图会影响搜索引擎的索引效率和覆盖率。
修复方案：创建 robots.txt（允许所有爬虫）和 sitemap.xml（列出所有页面 URL，包括中英文版本）。提交到 Google Search Console 和 Bing Webmaster Tools。
PWA 缺少 Web App Manifest 和多尺寸 favicon

修复方案：使用 favicon 生成工具（如 realfavicongenerator.net）生成全套 favicon，添加 <link rel="manifest">、<link rel="apple-touch-icon"> 等标签。
